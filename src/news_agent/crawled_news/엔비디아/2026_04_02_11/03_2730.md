AMD가 MLPerf Inference 6.0 제출 결과를 통해 생성형 AI 추론 인프라 경쟁력 강화에 나섰다. 이번 제출에서 AMD는 단순히 기존 벤치마크의 성능 수치를 높이는 데 그치지 않고, 신규 워크로드 확대, 멀티노드 환경에서 초당 100만 토큰 돌파, 파트너사를 통한 재현성 확보, 이기종 GPU 구성 검증까지 제시하며 추론 플랫폼 전반의 경쟁력을 강조했다.

AMD 설명에 따르면 이번 MLPerf Inference 6.0 결과는 고객들이 추론 플랫폼을 평가할 때 단일 지표만 보지 않는다는 점에 초점을 맞췄다. 단일 노드 성능, 대규모 확장 효율, 신규 모델 대응 속도, 파트너 시스템에서의 재현성, 소프트웨어 스택의 민첩성까지 종합적으로 입증하는 데 의미를 뒀다는 설명이다.

MI355X, 추론 중심 설계 강조

이번 제출의 중심에는 AMD Instinct MI355X GPU가 있다. MI355X는 AMD CDNA 4 아키텍처와 3nm 공정을 기반으로 1850억 개 트랜지스터를 집적했고, FP4와 FP6를 지원하며 최대 288GB HBM3E 메모리를 탑재한 것이 특징이다. AMD는 이를 통해 최대 5200억 파라미터 규모 모델을 단일 GPU에서 지원할 수 있다고 설명했다. 공랭식과 직접 액체 냉각 방식의 UBB8 노드 구성을 모두 제공하는 점도 배치 유연성 측면에서 내세웠다.

성능 측면에서는 최대 10페타플롭스급 FP4·FP6 연산 성능을 기반으로 대형 모델 수용력과 배포 준비성을 동시에 확보했다고 AMD는 밝혔다. 이는 단순 처리 속도뿐 아니라 실제 추론 서비스 환경에 필요한 플랫폼 성숙도를 겨냥한 설계라는 설명이다.

100만 토큰 장벽 돌파

이번 결과에서 가장 주목되는 지점은 AMD가 처음으로 MLPerf Inference에서 초당 100만 토큰을 넘어섰다는 점이다. AMD는 멀티노드 환경에서 Llama 2 70B의 Server와 Offline, GPT-OSS-120B의 Offline에서 이 기준을 돌파했다고 밝혔다. AMD는 이를 통해 클러스터 단위에서의 집계 처리량과 서비스 준비도를 입증했다고 평가했다.

Llama 2 70B 멀티노드 구성에서는 11개 노드, 87개 MI355X GPU를 활용해 Offline 104만2110토큰/초, Server 101만6380토큰/초, Interactive 78만5522토큰/초를 기록했다. 확장 효율은 Offline 93%, Server 93%, Interactive 98%로 제시됐다. GPT-OSS-120B의 경우 12개 노드, 94개 MI355X GPU 구성에서 Offline 103만1070토큰/초, Server 90만54토큰/초를 기록했고, 확장 효율은 각각 92%, 93%였다. AMD는 이를 통해 파일럿 수준을 넘어 생산 환경으로의 확장 가능성을 제시했다고 밝혔다.

이전 세대 대비 3.1배 향상

세대 간 성능 향상 폭도 주요 포인트다. AMD는 Llama 2 70B Server 기준으로 MI355X가 초당 10만282토큰을 기록해 이전에 제출했던 MI325X의 3만2028토큰 대비 3.1배 높은 처리량을 보였다고 설명했다. AMD는 CDNA 4 아키텍처, 높은 연산 밀도, FP4·FP6 지원, 대용량 HBM3E, ROCm 소프트웨어 최적화가 복합적으로 반영된 결과라고 분석했다.

엔비디아 B200·B300과의 비교도 제시됐다. Llama 2 70B 단일 노드 기준으로 AMD는 B200 대비 Offline 동률, Server 97%, Interactive 119% 수준의 성능을 기록했다고 밝혔다. B300과 비교해서는 Server 93%, Offline 92%, Interactive 104%로 설명했다. AMD는 특정 단일 시나리오가 아니라 Offline, Server, Interactive 전반에서 경쟁력을 보였다는 점을 강조했다.

신규 모델 대응도 확대

GPT-OSS-120B는 이번 MLPerf에서 처음 도입된 워크로드다. AMD는 첫 제출 모델임에도 불구하고 단일 노드 기준 B200 대비 Offline 111%, Server 115% 성능을 기록했다고 밝혔다. B300 대비로는 Offline 91%, Server 82% 수준을 나타냈다. AMD는 신규 모델을 빠르게 구동하고 최적화·정확도 검증·성능 확보까지 이어간 점을 의미 있는 성과로 제시했다.

텍스트-투-비디오 분야로의 확장도 포함됐다. AMD는 Wan-2.2-t2v를 통해 대형언어모델을 넘어 멀티모달·생성형 비디오 워크로드까지 범위를 넓혔다고 설명했다. 이번 제출에서는 Single Stream 시나리오에 집중해 Open 카테고리로 제출했으며, 공식 결과는 B200 단일 노드 대비 93%, B300 대비 87% 성능으로 제시됐다. AMD는 제출 마감 이후 추가 튜닝을 통해 Single Stream은 B200 대비 108%, B300과 동률 수준, 비공식 Offline 결과는 B200 대비 111%, B300 대비 88%까지 개선됐다고 덧붙였다. 다만 이 수치는 공식 MLPerf 제출 결과나 MLCommons 검증 대상은 아니라고 명시했다.

9개 파트너 참여, 4개 GPU 세대 확장

생태계 측면에서는 9개 파트너가 AMD Instinct 하드웨어 기반으로 제출에 참여했다. 참여사는 Cisco, Dell, Giga Computing, HPE, MangoBoost, MiTAC, Oracle, Supermicro, Red Hat 등이다. 제출은 MI300X, MI325X, MI350X, MI355X 등 4개 GPU 유형에 걸쳐 이뤄졌다. AMD는 이를 통해 특정 플래그십 구성에 국한되지 않는 OEM·ODM·클라우드형 플랫폼 전반의 확장성을 보여줬다고 설명했다.

특히 MI355X 기반 파트너 제출 결과가 AMD 자체 제출 수치 대비 4% 이내에 들어왔고, 일부는 1% 이내에 근접했다고 밝혔다. AMD는 이를 실험실 수준 수치가 아니라 실제 파트너 시스템에서도 재현 가능한 결과라는 신호로 해석했다. 실제 배포 위험을 줄이고 생산 환경 전환 속도를 높일 수 있다는 점도 함께 제시했다.

미국-한국 분산 이기종 GPU 구성도 제시

이번 제출에서는 첫 3-GPU 이기종 MLPerf 제출 사례도 포함됐다. Dell과 MangoBoost가 제출한 해당 구성은 MI300X, MI325X, MI355X를 함께 활용해 Llama 2 70B Server에서 14만1521토큰/초, Offline에서 15만1843토큰/초를 기록했다. AMD는 이 구성이 단순한 혼합 세대 운영을 넘어 서로 다른 지역의 시스템을 오케스트레이션했다는 점에서 의미가 있다고 설명했다. 실제로 MI355X 플랫폼은 미국 Dell 연구실에, MI300X와 MI325X 플랫폼은 한국에 위치한 상태에서 구성됐다고 밝혔다.

AMD는 이를 통해 기존 Instinct GPU 구축 환경을 전면 교체하지 않고 연장 활용할 수 있으며, 세대가 다른 하드웨어도 효율적으로 연계 운영할 수 있다는 가능성을 제시했다. ROCm 소프트웨어가 비균일 환경에서의 워크로드 분산과 운영 실용성을 높이는 역할을 한다는 설명도 덧붙였다.

ROCm과 연간 로드맵으로 장기 전략 부각

AMD는 이번 제출 전반을 관통하는 핵심 요소로 ROCm 소프트웨어를 제시했다. FP4 실행 최적화, 멀티노드 GPU 간 통신 최적화, 이기종 환경 워크로드 분배, Llama·Wan·GPT-OSS 등 신규 모델 대응이 모두 ROCm 기반으로 구현됐다고 설명했다. AMD는 ROCm이 단일 벤치마크 결과를 위한 보조 수단이 아니라 성능, 확장성, 유연성, 재현성을 뒷받침하는 기반이라고 규정했다.

장기 로드맵도 함께 제시됐다. AMD는 2023년 MI300X, 2024년 MI325X, 2025년 MI350 시리즈와 MI355X로 이어지는 연간 출시 주기를 강조하며, 2026년에는 차세대 CDNA 5 아키텍처 기반 MI400 시리즈와 Helios 랙 스케일 AI 솔루션으로 확장할 계획이라고 밝혔다. 이를 통해 단기 성능 경쟁을 넘어 대규모 AI 추론 인프라의 지속적 진화를 겨냥하고 있다는 메시지를 내놨다.
